Javascript의 자료형

Javascript는 6가지의 Primitive 데이터 타입과
Object 타입의 자료형을 사용한다.(총 7가지 Type)

Primitive 타입으로는

  • Boolean Type
  • Null Type
  • Undefined Type
  • Number Type
  • String Type
  • Symbol Type
    이 있고, 그외의 복합적인 자료형으로 Object Type이 따로 있다.

Primitive Type

Primitive타입은 원시 자료형이라고 하며
값이 유한하고 Immutable(불변함)을 따른다.
또한, 데이터간의 비교가 가능하다.

Boolean은 논리요소로 true 와 false로 값을 표현한다.
Null은 말 그대로 Null값으로 빈 값(없는 값)
undefined 는 변수 선언 후 값이 없을 때의 값으로 null과 undefined는 다른것으로 취급해야 함
Number는 다른 프로그래밍언어와 달리 정수, 소수가 다 포함된 숫자의 자료형으로 NaN과 +-Infinity라는 특별한 값을 가지기도 한다.
String타입은 문자열 데이터를 나타내는데 사용하며 +연산자를 이용해 문자열끼리를 합칠 수 있다.
Symbol타입은 ECMAScript 6 에서 새로 추가된 타입으로
고유한 특정 값을 말한다. Symbol을 생성하면 고유함을 보장하기 때문에 이름의 충돌과같은 문제를 막을 수 있다.

Object Type

Object타입은 Primitive 타입을 제외한 나머지 자료형을 말한다.
Array, Date, Object, RegExp 등과 같은 자료형들은 모두 Object타입이며 Function도 마찬가지로 Object 타입이지만 typeof함수로 검사했을때 특이하게도 function으로 표시된다.(이유는 저도..모릅니다..ㅠㅠ)
모든 Object타입은 Key, Value값을 통해 나타낼수 있고,
데이터의 모양을 무한하게 만들 수 있다.
또한 같은 값의 배열, 오브젝트라도 각각의 저장된 메모리가 다르기 때문에 데이터 비교를 할 수 없다.
